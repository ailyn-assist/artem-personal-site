Политика конфиденциальности расширения «Ответка»

Версия: 1.2

Действует с: 8 мая 2026 г.

Применимо к: Chrome-расширению «Ответка — AI-ответы на отзывы» (далее — «расширение»), распространяемому через Chrome Web Store.

Этот документ описывает, какие данные обрабатывает расширение, как они собираются, обрабатываются, хранятся и передаются. Документ соответствует требованиям Chrome Web Store Developer Program Policies, в том числе политике Limited Use of User Data.

Коротко: у разработчика нет серверов, через которые проходили бы пользовательские данные. API-ключ и настройки хранятся только локально в браузере пользователя. Тексты отзывов передаются напрямую выбранному пользователем AI-провайдеру (OpenAI, OpenRouter или DeepSeek) по ключу пользователя и для других целей не используются.

1. Кто отвечает за расширение

Расширение разработано и поддерживается Артёмом Бандиным (далее — «разработчик»).

Разработчик не является оператором персональных данных в строгом смысле: у него нет собственных серверов или backend-инфраструктуры, через которую проходили бы данные пользователя. Все настройки и API-ключи хранятся локально в браузере пользователя; тексты отзывов передаются напрямую выбранному AI-провайдеру по ключу самого пользователя, минуя инфраструктуру разработчика.

Разработчик отвечает за функциональность расширения, дизайн обработки данных и соблюдение этой политики.

2. Какие данные собираются

Расширение работает с тремя типами данных:

  1. API-ключ AI-провайдера (данные аутентификации). Вводится пользователем один раз в настройках расширения. В зависимости от выбора пользователя это ключ от одного из трёх провайдеров: OpenAI, OpenRouter или DeepSeek.
  2. Настройки расширения. Выбранный провайдер, выбранная модель, системный промпт, параметры генерации (temperature, max_tokens), лимит символов для дополнительного контекста.
  3. Содержимое страницы Яндекс.Бизнеса. Текст конкретного отзыва, имя автора отзыва, оценка и (по желанию пользователя) дополнительный контекст со стороны бизнеса, который пользователь сам вводит в модалке «С контекстом».

Расширение не собирает:

  • персональные данные пользователя расширения (имя, email, телефон, адрес — расширение не имеет регистрации и аккаунтов);
  • финансовые и платёжные данные;
  • данные о местоположении, IP-адрес, GPS-координаты;
  • историю просмотров, поисковые запросы, cookies, идентификаторы устройства;
  • любые данные с других сайтов, кроме страниц отзывов Яндекс.Бизнеса.

3. Как данные обрабатываются (цели и основания)

Данные обрабатываются строго в рамках единственного назначения расширения — генерация ответов на отзывы клиентов в Яндекс.Бизнесе.

Тип данныхЦель обработки
API-ключ AI-провайдера Аутентификация пользователя у выбранного провайдера при генерации ответа и при проверке валидности ключа на странице настроек.
Настройки расширения Применение пользовательских предпочтений к запросу в AI-API (провайдер, модель, тон, лимиты).
Текст отзыва, имя автора, оценка Передача в API выбранного провайдера для генерации текста ответа.
Доп. контекст от бизнеса (опционально) Учёт фактов со стороны бизнеса при генерации ответа.

Все типы данных используются только для целей, описанных выше. Расширение не использует и не передаёт эти данные:

  • для рекламы или таргетинга;
  • для определения платёжеспособности или кредитного скоринга;
  • для целей, не связанных с основной функциональностью расширения;
  • для обучения моделей машинного обучения со стороны разработчика расширения.

Это соответствует требованиям политики Limited Use of User Data Chrome Web Store.

4. Как данные хранятся

Тип данныхГде хранятсяСрок хранения
API-ключ AI-провайдера Локально в браузере пользователя (chrome.storage.local). До удаления пользователем или до удаления расширения.
Настройки расширения Локально в браузере пользователя (chrome.storage.local). До удаления пользователем или до удаления расширения.
Текст отзыва, имя автора, оценка Не хранятся. Данные считываются с открытой страницы Яндекс.Бизнеса в момент нажатия кнопки и удерживаются в оперативной памяти только на время одного запроса к AI-провайдеру. Не хранятся расширением.
Доп. контекст от бизнеса Не хранится. Вводится пользователем в модалке и удерживается в памяти только на время одного запроса. Не хранится расширением.

На серверах разработчика расширения данные не хранятся — у расширения нет своих серверов и backend-инфраструктуры.

Меры безопасности:

  • Данные в chrome.storage.local изолированы Chrome от других расширений и других сайтов; доступ к ним есть только у самой «Ответки».
  • Запросы к AI-провайдерам выполняются исключительно по HTTPS (TLS).
  • API-ключ передаётся только в заголовке Authorization HTTPS-запроса к выбранному провайдеру и больше никуда.
  • Расширение не использует удалённый код (eval, внешние <script src>, динамические импорты), что исключает классы атак с подменой исполняемого кода.

5. Кому данные передаются

Расширение передаёт данные одному выбранному пользователем AI-провайдеру из трёх возможных. Передаётся только тому провайдеру, чей API-ключ пользователь сохранил в настройках на момент запроса. Данные не дублируются между провайдерами.

ПолучательХостКакие данныеКогдаЗачем

Генерация текста ответа на отзыв; проверка валидности ключа.

Обработка данных у каждого провайдера подчиняется его собственной политике конфиденциальности:

  • OpenAI: openai.com/policies/privacy-policy, openai.com/policies/api-data-usage-policies
  • OpenRouter: openrouter.ai/privacy, openrouter.ai/terms
  • DeepSeek: platform.deepseek.com/privacy

Перед выбором провайдера пользователю рекомендуется ознакомиться с политикой соответствующей компании. В частности, OpenRouter и DeepSeek могут иметь иные требования к хранению логов запросов и территории обработки данных, чем OpenAI.

Расширение не передаёт данные:

  • разработчику расширения (нет серверов разработчика);
  • рекламным сетям;
  • аналитическим сервисам (Google Analytics, Яндекс.Метрика и т. п. в расширении не используются);
  • любым другим третьим лицам, кроме выбранного пользователем AI-провайдера.

Расширение не продаёт пользовательские данные.

6. Права пользователя

Пользователь имеет право:

  • Доступ: в любой момент посмотреть, какие настройки сохранены, через страницу настроек расширения.
  • Изменение: отредактировать любую настройку, включая API-ключ и провайдера, на странице настроек.
  • Удаление: очистить все локальные данные либо вручную (стереть API-ключ и сохранить), либо удалив расширение через chrome://extensions/ — Chrome автоматически очистит chrome.storage.local.
  • Отзыв согласия: прекратить использование расширения в любой момент. Отозвать согласие на обработку у AI-провайдера можно, удалив API-ключ из настроек.
  • Удаление данных у AI-провайдера: обратиться напрямую к выбранному провайдеру (OpenAI, OpenRouter или DeepSeek) согласно его политике.

7. Дети

Расширение не предназначено для лиц моложе 16 лет и сознательно не собирает данные о детях. Если вам стало известно о таких данных, сообщите разработчику — они будут удалены.

9. Соответствие Chrome Web Store Developer Program Policies

Разработчик расширения подтверждает:

  • Сбор и использование данных ограничены практиками, описанными в этой политике.
  • Использование данных соответствует политике Limited Use of User Data Chrome Web Store: данные используются исключительно для предоставления и улучшения основной пользовательской функции расширения; не используются для рекламы; не продаются; не передаются третьим лицам, кроме случаев, перечисленных в разделе 5; не используются для определения платёжеспособности.
  • Данные аутентификации (API-ключ AI-провайдера) обрабатываются с особой осторожностью: хранятся только локально, передаются только тому провайдеру, чей ключ ввёл пользователь, и только в зашифрованном HTTPS-канале.
